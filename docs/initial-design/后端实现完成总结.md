## 技术亮点

### 1. 完整的 RAG 流程
- 视频内容读取 → 文本切分 → 向量化 → 存储 → 检索 → 生成

### 2. 用户数据隔离
- 所有业务表包含 user_id
- 向量ID包含用户信息（userId:bvid:chunkIndex）
- 检索时按用户过滤

### 3. 级联删除策略
- 删除视频时自动清理：DashVector向量 → 会话 → 消息 → 分片 → 映射

### 4. 流式响应
- 使用 SSE (Server-Sent Events) 实现流式对话
- 异步处理，不阻塞主线程

### 5. 事务管理
- 关键操作使用 @Transactional 保证数据一致性

## 注意事项

### 1. DashVector 配置
- 需要在 DashVector 控制台创建名为 `rag_bilibili` 的 Collection
- 向量维度需要与 Embedding 模型匹配

### 2. B站凭证
- SESSDATA、bili_jct、buvid3 不能落库
- 仅在前端保存，每次导入视频时传递

### 3. 向量删除
- DashVector 仅支持基于 ID 的删除
- 不支持条件删除

### 4. 性能优化
- 批量插入使用 MyBatis foreach
- 向量检索使用 TOP_K 限制
- 聊天历史限制最近10轮
